# ---- Sanity checks on dNBR threshold fields --------------------------------
# These checks guard against the most common misuse: treating threshold values
# (dNBR index units, typically 50–600) as if they were percentages (0–100).

valid_thresh <- mtbs_west |>
  st_drop_geometry() |>
  filter(thresholds_valid)

# Check 1: Threshold orientation.
# MTBS fires can be mapped on either axis of the dNBR spectrum:
#   "Burned" fires:             low_t < mod_t < high_t (thresholds increase toward high severity)
#   "Increased Greenness" fires: low_t > mod_t > high_t (thresholds decrease, negative dNBR)
# Both orderings are valid. What is NOT valid is a MIXED ordering (non-monotone),
# which would indicate corrupted threshold values.
valid_thresh <- valid_thresh |>
  mutate(
    thresh_orient = case_when(
      low_t < mod_t & mod_t < high_t ~ "burned",       # typical positive-dNBR fire
      low_t > mod_t & mod_t > high_t ~ "greenness",    # increased-greenness fire
      TRUE                            ~ "mixed"         # non-monotone — data problem
    )
  )

orient_tbl <- valid_thresh |> count(thresh_orient)

Plot 4 — Burn Severity Threshold Distributions

The MTBS perimeter shapefile stores fire-specific dNBR threshold values, not per-class area percentages. Three plots describe what we can characterize from these fields:

  • 4a — Distribution of High_T (Moderate→High boundary). Fires with a lower High_T value have more pixels classified as high severity (the dNBR bar is set lower).
  • 4b — Distribution of dNBR_offst (mean dNBR in unburned reference pixels outside the perimeter). Larger positive values indicate greater pre→post spectral change in the surrounding landscape.
  • 4c — High_T vs. fire size: whether larger fires tend to have systematically different severity thresholds.
Important

Per-class area breakdowns (% low / % moderate / % high) are not available in the perimeter shapefile. These require the individual fire severity rasters distributed separately by MTBS (one raster per fire). Download and process those rasters before reporting severity area statistics.
